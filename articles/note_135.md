# 【AI対話アプリ】200の問い #135「公務員・論理的報告」

このアプリは「公務員・論理的報告」のためのAI対話プロンプト集です。上司の心を掴む「報告書」と「口頭報告」の論理構造を、AI分析で言語化し、信頼と昇進を勝ち取るツールです。

---


## 1. 「データ」ではなく「判断命題」から始める

公務員の報告書は通常「データ」で始まります。「令和3年度のX事業の実績は…」。上司が必要なのは「データ」ではなく「判断命題」です。AIに「この30ページの報告書の『最初の1ページ』は何を書くべきか。『上司が判断すべき選択肢は何か』という『問い』から始める報告書に」と聞きます。その判断命題が「上司の注意」を掴みます。


## 2. 「肯定的なデータ」と「否定的なデータ」の両立を示す

信頼できる報告書は「都合の良いデータだけ」を示しません。AIに「この事業について『成功を示すデータ』『懸念を示すデータ』の両方を提示してください。そして『懸念に対して、あなたはどう対策するのか』を説明してください」と聞きます。バランスの取れた報告が「この職員は信頼できる」という印象を与えます。


## 3. 「選択肢と推奨」の2段階構成で上司の判断を導く

報告書が「◯◯すべき」と一択しか示さないと「部下の一方的な主張」に見えます。AIに「この事業について『選択肢A』『選択肢B』『選択肢C』の3つを提示した上で『私はAを推奨する理由は』という構成に再構成してください」と聞きます。選択肢を提示した上での推奨が「部下の提案」に見えます。


## 4. 「口頭報告での30秒プレゼン」を用意する

書面の報告書より「口頭での30秒」が上司の判断を決めます。AIに「この事業の『現状』『課題』『推奨アクション』『期待される結果』を30秒以内に説明するスクリプトを」と聞きます。その30秒が「この事業の本質」を表現します。


## 体験談: 「30ページのデータ報告書を『上司の判断を促す論理構造』に変えて、昇進評価が変わった」

正直に書きます。昨年の報告では「データがある=判断ができる」と単純に考えていました。上司から「で、結論は?」と聞かれたのは「判断命題がない」ことを指摘されていたのです。今回、AIに報告書の論理構造を再構成させました。最初のページを「この事業は、継続か中止か、あるいは規模縮小か。その判断に必要なデータがある」という『判断命題』に変えました。次に「事業成功を示すデータ」「事業の懸念を示すデータ」の両方を並列しました。最後に「3つの選択肢」と「なぜAを推奨するのか」を示しました。口頭報告では「30秒で『現状・課題・推奨・期待結果』」をスクリプト化して臨みました。上司の反応は「わかりやすかった。この方針で進めてくれ」と即座に判断してくれました。その3ヶ月後、昇進の面接で「判断が早い人」として評価されました。昇進は「データの量」ではなく「判断を促すロジック」の質だったのです。

## 🎯 上級プロンプト: 「報告書の『判断命題テンプレート』」法

報告書が「上司の判断を促さない」と感じたら、AIに次のように頼みます:
「この報告書の『第1ページ』を『上司はこの事業について何を判断すべきか。その判断に必要な情報は何か。その情報を3つの選択肢として提示した上で、あなたの推奨は何か』という構成に再構成してください。」

```
あなたは、公務員が上司へ行う報告書作成および口頭報告において、論理的思考と説得力を高める専門コーチです。上司の理解と信頼を得るための報告術に特化し、あなたの報告内容を具体的なステップで改善する支援をします。

私があなたの報告について分析し、改善提案を行うための対話を始めましょう。まず、以下の質問に答えてください。
1. あなたが上司に報告したいと考えている件名と、その報告の最も重要な目的を具体的に教えてください。
2. この報告の種類は「書面報告」と「口頭報告」のどちらが主となりますか。また、対象の上司はどのようなタイプの方ですか。（例：結論を重視する、経緯を重視する、多忙、慎重派など）

これらの情報を踏まえ、私はあなたの報告の現状についてさらに質問します。私の質問に順次答えていく形で、報告の骨子、伝えたいポイント、懸念事項などを詳しく教えてください。対話を通じて、最適な報告の論理構造と表現方法を一緒に見つけ出しましょう。

一連の対話と分析が完了した後、あなたの報告がより上司に響くための具体的な改善策を提示し、最後に、あなたが今日から実行できる「具体的な次の一歩」を3つ提案してこのセッションを終了してください。
```

この問いにより、報告書が「データ提示」から「判断促進」へ変わります。
