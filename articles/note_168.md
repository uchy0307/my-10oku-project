# 【AI対話アプリ】200の問い #168「通知書・下書き構成」

このアプリは「通知書・下書き構成」のためのAI対話プロンプト集です。会社から顧客へ送る『報告』『謝罪』『提案』の通知書を、書き手の「感情」から「顧客の行動」まで計算した構成で言語化し、一目で読まれる書簡へと変えます。

---


## 1. 「伝える内容」を『事実』『影響』『対応』『謝罪』の4層に分解する

通知書が読みにくい理由は、構成が曖昧だからです。AIに「この納期遅延について、『いつ遅れたのか（事実）』『顧客にどう影響するのか（具体的な被害）』『今後どう対応するのか（提案）』『なぜそうなったのか（謝罪の深さ）』の4層で、各100字で構成を作成してください」と依頼します。その4層が明確に分かれることで、読み手は「この企業は何を言っているのか」が一瞬で理解できます。


## 2. 「顧客の立場」を想像して、被害を具体化する

「ご不便をおかけして申し訳ございません」という一般論では心に届きません。AIに「この納期遅延により、顧客の現場では『実際に何が困るのか』。その顧客の立場で『困りの具体シーン』を3つ、その顧客の言葉で200字で書いてください」と依頼します。例えば「製造ラインが1日止まる」という抽象的な影響ではなく「現場の10人の作業者が、予定していた仕込みを延期するしかなく、明日の納期に間に合わなくなる」という具体化です。


## 3. 「これからの信頼回復」を『行動』で示す

謝罪だけでは信頼は戻りません。AIに「今回のような遅延を『二度と起こさないために』、この企業が『今週中』『来月中』『今年中』で『実施する具体的な対応』を3つ、各100字で」と依頼します。「体制を見直します」という曖昧さではなく「毎日朝8時にチェック体制を組む」という具体的な行動です。


## 4. 「署名」を『個人』にする

企業の代表名での署名より、実務者の名前で署名する方が信頼が高まります。AIに「この通知書の最後に『実務担当者からのメッセージ』を100字で付け足してください。そこには『実務担当者の個人的な謝罪』と『顧客との信頼を取り戻すための決意』を含める」と依頼します。


## 体験談: 謝罪メール1通で『3年の取引継続』が決まった話

その謝罪メールは、上司の想定より長くなりました。構成は『事実→被害具体化→今週の対応→来月の対応→実務者の個人的メッセージ』で計800字。上司は「長すぎる」と言いましたが、顧客の立場を想像し、その手間をかけることにした。送信から3日後、顧客から返信が来ました。「こういう謝罪は初めてだ。何をしたのか、なぜそうなったのか、これからどうするのかが、全部わかった」という返信です。その後、取引は継続されるだけでなく、顧客は別の製品の発注まで追加してくれました。3年後、その顧客の営業責任者は「あの謝罪メールがあったから信頼できた」と言ってくれました。

1. この通知により、顧客の『現場で何が困るのか』を具体的に言えるか
2. その困りを『顧客の言葉』で表現できるか
3. 今この瞬間、何をすれば信頼が戻るか
4. 来月までに、何を『形に』してみせるのか
5. 1年後、顧客はこの企業について何と言っているか

## 🎯 上級プロンプト: 「顧客の返信メール」法

通知書の構成が迷ったら、AIに次のように頼みます:
「この企業からの通知メールを受け取った顧客が『最高の返信メール』を書くとしたら、何と言うか。その返信から『この企業は何をしたから信頼が戻ったのか』を逆算して」

この顧客の返信メールが見えると、通知書に何が足りないか明確になります。
