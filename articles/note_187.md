# 【AI対話アプリ】200の問い #187「小説・伏線の種まき」

このアプリは「小説・伏線の種まき」のためのAI対話プロンプト集です。「意外性のある結末」を書きたいのに「唐突に見える」という読者コメントを受けるのは、伏線の「埋め込み方」が計算されていないからです。一読目で「気づかれない」が「二読目で『ああ、あそこか!』と腑に落ちる」伏線の設計を言語化します。

---


## 1. 「犯人決定」から「逆算した伏線リスト」を作る

長編で意外な真犯人を設定したら、まずAIに「この人物が『実は犯人だった』と二読目に気づく読者のために、埋めるべき伏線は何か、複数の『層』に分けて列挙してください」と頼みます。例えば真犯人が「一見善人の町内会長」なら、伏線は「①目線の層(町内会長だけが〇〇を知っていた)」「②セリフの層(一言だけ『本当は』を示唆する言葉)」「③行動の層(他の人物には取らない『その行動』だけを取る)」という複数層に分かれます。各層が満たされると「結末を知った読者が二読すると『あ!』と複数の箇所で気づく」という快感が生まれます。


## 2. 「目立たないシーン」に「決定的なディテール」を埋める

伏線は「主要な場面」では気づかれやすいので「脇役のセリフ」「風景描写」「偶然のように見えるシーン」に埋めます。AIに「第3話で、真犯人が『町内会の資料整理』という何でもない場面にいるとしたら、その場面に『犯人を匂わせるが、一読目では見落とされる』ディテールを3つ埋めてください」と頼みます。例えば「本来はないはずの古い帳簿を持っている」「その帳簿を誰かに見られないようにさっと隠す」「その瞬間の『目の色』が普段と違う」——これらは「何でもない日常シーン」にこっそり埋め込まれます。一読目では「ただの世話焼き」で済みますが、二読目には「あ、この人が隠していた」と繋がります。


## 3. 「セリフの一行」で「本当の心理」をチラ見せする

最も効果的な伏線は「セリフ」です。真犯人が「一見無意識に」発する言葉の中に「本当の知識」や「本当の感情」が垣間見える。AIに「第8話で、町内会長が他の登場人物と『30年前の失踪事件』について話すシーンがあるとしたら、会長が『つい本当のことを言いかけるが、すぐ言い直す』というセリフを1行設計してください。一読目では『変な人』だが、二読目では『あ、実は知ってた』と見える台詞」と頼みます。例えば「『あの日、誰かが……いや、警察によれば無く人は』」というように、口をついて出る「本当」が一瞬見えて、すぐ蓋をする。その一行が、結末を知った読者の「ああ!」を産みます。


## 4. 「複数の伏線」が「終盤で一気に繋がる構成」

伏線は「単一」では弱いです。複数の伏線が「終盤で一気に繋がり、それまで別々に見えていた点が一つの図形になる」という快感が必要です。AIに「第1話での『ディテール伏線』『第5話でのセリフ伏線』『第12話での行動伏線』という3つの異なる層の伏線が、最終話の『真犯人判明シーン』で『全部、同じ矢印を指していた』と読者が気づく構成を設計してください」と頼みます。点が繋がると、読者は「あ、この作者は全て計算していたんだ」と感動します。


## 体験談: 「唐突な真犯人」から「計算された意外性」へ

初稿では、真犯人が「終盤で突然出てきた」ように見えました。編集者に「この人物は序盤から『本当は何か知ってる人』に見せましたか?」と聞かれ、返す言葉がありませんでした。再稿で、AIの指導を受けて「第1話での『古い帳簿の一瞬の反応』」「第5話での『一言のセリフ』」「第12話での『その手の動き』」という3層を埋め直しました。完成稿を読んだ編集者は「二読したら『ああ、ここか!』が3回ある。良いね」と言いました。最初は「全部の伏線を用意する方が親切」だと思っていました。実は「気づかれないが、二読で『ああ!』となる伏線」の方が、読者の心に残る。それを学びました。

## 🎯 上級プロンプト: 「二読目の『ああ!』マップ」法

伏線の埋め込みで迷ったら、AIに次のように頼みます:
「結末を知った読者が、物語を二読するとき『ああ、そういうことだったんだ』と気づく瞬間を『5箇所』リストアップしてください。各箇所で『何に気づくか』を描写して。」

このプロンプトで、伏線の埋め込みが「計画的」になります。私はこれで「二読目で読者が『5つの瞬間』で気づく快感」を意識的に設計できるようになり、単なる「結末の驚き」ではなく「再発見の喜び」を作る作家になりました。
