# 【AI対話アプリ】200の問い #199「eスポーツチーム分析」

このアプリは「eスポーツチーム分析」のためのAI対話プロンプト集です。ゲームの「勝敗」は「キャラクターの強さ」でも「プレイヤーのスキル」でもなく「チームの『メンタル構造』『信頼の質』『ストレス下での判断』」で決まります。デジタル時代の「勝敗の本質」を言語化します。

---

## 1. 「ゲーム内の判断」が「実は『メンタル状態』に支配されている」ことを見える化する

eスports選手は「自分は論理的に判断している」と思います。実際は「その判断は『ストレス下での不安』『信頼の有無』『プレッシャーの質』」に支配されています。AIに「あなたのチームが『負けた試合』での『判断ミス』を1つ挙げて、その『判断ミス』の奥に『何のメンタル状態』があったか。①チーム内の信頼不足 ②個人の不安 ③指示への疑い」と頼みます。例えば「エース選手の『攻めるべき場面で守った』という判断ミスは『実は『他の選手への不信』『自分だけで勝つしかないという焦り』」から来ていたことに気づく。その気づきが「対策」を生みます。

## 2. 「試合前の『メンタルチェック』」を「数値化」する

勝つチームは「試合直前に『メンタル状態』を『自覚させる』」という工夫をしています。AIに「試合30分前に『あなたのチーム5人に『今、何が怖いですか』『チームの『誰を信頼していますか』『自分は『何ができますか』」という『3つの問い』を投げたとき『その答えから『チームの『メンタル準備度』を『点数』で判定』してください」と頼みます。「全員が『信頼している』『自分の役割が見えている』」という答えが並ぶと「メンタル準備度80点」。「誰かが『不安』『役割が曖昧』」という答えが出ると「メンタル準備度40点」という形で「試合前のメンタル状態」を見える化できます。

## 3. 「『ストレス下での判断パターン』」を事前に『共有』する

同じプレイヤーでも「普通の状態」と「ストレス下」では『全く違う判断』をします。AIに「あなたのチームの『各選手が『ストレス下で『どんな判断の癖』を持っているか』を『5パターン』リストアップしてください。そして『その癖を『試合中に気づける『チーム内の合図』」を設計してください」と頼みます。例えば「エース選手は『不安になると『無理な攻めに転じる』という癖→試合中に『エースが焦っているな』と見えたら『サポート選手が『守りを固める指示』を出す』」という「メンタルバランスの調整」が生まれます。

## 4. 「勝敗の『本当の分析』」は「ゲーム内のデータ」ではなく「試合前後の『メンタル変化』」

多くのチームは「敗因＝キルデス数」「勝因＝ダメージ量」と分析します。それでは次は変わりません。AIに「今日の『敗戦の本当の原因』は『ゲーム内のデータ』ではなく『試合の『どこで』『誰の『メンタルが揺らいだか』』を特定してください」と頼みます。「第2ラウンド中盤、エース選手の『攻撃が通らない』という出来事→『あ、自分の判断が間違ってるのかもしれない』という『疑い』」→「その疑いが『次の判断を『迷わせた』」という「メンタルの連鎖」が見える。その「メンタル崩壊の瞬間」に気づく監督だけが「次の対策」を立てられます。

## 体験談: 「『データ分析』から『メンタル分析』へ」

初年度、私は「敗戦の分析＝ゲーム内のデータ解析」をしていました。「キルが少ない」「ダメージ効率が低い」——全部「数字の改善」を目指していました。それでも「試合では『同じミスが繰り返される』」。ターニングポイントは、試合後のメンバー面談で「実は『試合の中盤で『このメンバーを信じていいのか』という疑いが生まれた』」というコメント。その瞬間、気づきました。「勝敗の『本当の原因』は『メンタル層』なんだ」。その後、試合前の「メンタルチェック」を導入し「試合中の『メンタル変化』を監視」「敗因分析＝『メンタル崩壊の瞬間』の特定」に切り替えました。結果「同じプレイヤーで『試合成績が向上』」。eスports は「スキル」ではなく「信頼」が全てなんだと分かりました。
